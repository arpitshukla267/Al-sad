## Bifurcation Patterns

### 1. Category-Specific Filters
- **Architectural Hardware**: Door-specific filters (Door Weight, Type)
- **Structural Materials**: Dimensional filters (Thickness, Width, Length)
- **Tools**: Type-based filters (multiple type filters)
- **Industrial Materials**: Material and size-focused filters

### 2. Filter Inheritance
- **Material**: Appears at multiple levels
- **Finish**: Appears at product type level
- **Type/Brand**: Appear where relevant

### 3. Dynamic Filter Sets
- Filters change based on product type
- Some categories have more filters than others
- Filter complexity varies by product category

## Implementation Recommendations

### URL Structure
```
/products/[category]/[subcategory]/[product-type]
```

Example:
- `/products/architectural/architecture-hardware/hinges`
- `/products/structural/screws-nails/drywall-screw`
- `/products/professional-tools-gear/hand-tools/general-hand-tools`

### Filter System Architecture
- **Level 2**: Category-specific filter sets
- **Level 3**: Product-type-specific filter sets
- **Dynamic Filter Rendering**: Based on product type

### Data Model Structure
```javascript
{
  category: "architectural",
  subcategory: "architecture-hardware",
  productType: "hinges",
  filters: {
    type: [...],
    doorWeight: [...],
    material: [...],
    color: [...],
    brand: [...]
  }
}
```

### Filter Component Strategy
- Create filter configuration objects per product type
- Render filters dynamically based on current product type
- Support multiple filter types: dropdowns, checkboxes, range sliders

## Notes
- This structure allows for a flexible, scalable product catalog
- Filters are context-aware and change based on product category
- The hierarchy supports deep navigation while maintaining clear organization
